# Docker 核心概念与实战小抄

Docker 是一个强大的“软件沙盒”和“环境克隆机”，能让你在隔离的环境中运行各种软件、数据库和科研工具，完美解决“电脑环境被配坏”和“换台电脑代码跑不通”的痛点。

## 🐳 核心概念

- **镜像 (Image)**：静态的安装环境包。相当于包含了系统库、依赖项和软件的“只读模板”。
- **容器 (Container)**：镜像运行起来的实例。就像基于这个模板凭空变出来的一台“微型独立电脑”。

## 🛠️ 常用基础操作速查表

### 1. 查看类命令
- `docker --version`：查看 Docker 是否安装成功及版本。
- `docker ps`：查看**当前正在运行**的容器。
- `docker ps -a`：查看**所有**容器（包括已经运行结束或被停止的）。
- `docker images`：查看本地已经下载好的所有镜像。

### 2. 运行与启动类命令
- `docker run [镜像名]`：根据某个镜像启动一个新容器。
  **常用的组合参数：**
  - `-d`：(Detached) 让容器在后台静默运行，不卡住你的终端命令行。
  - `-p 宿主机端口:容器内端口`：端口映射（比如想让外部访问容器里的 80 端口网页，可以通过本机的 8080 进入，写成 `-p 8080:80`）。
  - `--name 名字`：为这个容器起一个方便记忆的名字。
  - `-it`：(Interactive TTY) 交互模式，通常配合后面加上 `bash` 使用，能让你直接“进入”容器的命令行系统查阅文件。

### 3. 清理与释放空间
- `docker stop [容器名或ID]`：温柔地让一个运行中的容器停下来。
- `docker rm [容器名或ID]`：把一个（或者多个）已经停止的容器**彻底删掉**。
- `docker rmi [镜像名或ID]`：把一个本地用不到的镜像安装包删掉，释放硬盘空间。

## 🚀 生信 / 数据科学三大实战场景场景

### 1. 秒开 Jupyter 数据科学全家桶
如果你不想在本地折腾 Python/R 环境，直接跑官方配置好的镜像：
```bash
docker run -p 8888:8888 jupyter/datascience-notebook
```
浏览器打开终端里提示的链接，直接就能写代码分析数据，内含 Python, R, Julia，各种包都已经装好了。

### 2. 测试 Web 服务应用 (Nginx)
瞬间部署一个网站：
```bash
docker run -d -p 8080:80 --name my-nginx nginx
```

### 3. 用后即弃的数据库环境
需要用 MySQL，但不想在电脑里配环境变量和系统服务？
```bash
docker run -d --name my-mysql -e MYSQL_ROOT_PASSWORD=your_password -p 3306:3306 mysql:latest
```
用完之后 `docker stop my-mysql && docker rm my-mysql` 渣都不剩。
